## Goal

Upgrade the existing AgriTrust Masumi agent so that every paid invocation returns a **signed assessment** with the five required fields, generated by an explainable AI reasoning step over the live Neo4j graph. Surface it inside the loan application review screen.

The Masumi discovery + payment + Cardano-style escrow flow already works end-to-end (`payAndInvoke`, `signReceipt`, `agent_jobs` table, `/api/public/agent/discover`, `/api/public/agent/invoke`). This plan only adds the reasoning layer and the new UI fields — no new secrets, no schema changes.

## What changes

### 1. Reasoning step — `src/lib/agritrust-agent.server.ts`
After assembling Neo4j + Supabase signals, call Lovable AI Gateway (`google/gemini-3-flash-preview`) with a structured `Output.object` schema and return a single payload:

```
{
  farmer_trust_score:        0–100,
  credit_risk_score:         0–100  (100 = highest risk),
  recommended_lending_limit_kes: number,
  confidence_level:          0–1,
  risk_explanation:          string (3–5 sentences, plain language),
  positive_factors:          string[],
  negative_factors:          string[]
}
```

The model is grounded in the existing `components`, Neo4j `graph_signals`, climate penalty, and farmer profile. Tier gating stays: `basic` returns score + risk + recommendation + explanation; `standard` adds identity + farm; `premium` adds graph signals.

Failure mode: if the gateway returns 429/402/5xx the call falls back to the existing rules-based recommendation and a templated explanation so the demo never breaks. 4xx errors surface to the lender UI.

### 2. Invocation path — no shape changes
`src/lib/agent.functions.ts` (`lenderRequestFarmerProfile`) and `src/routes/api/public/agent/invoke.ts` already call `buildAgritrustProfile` and return `{ invocation, profile, climate, receipt, signature }`. The receipt is extended to include the new fields so the HMAC signature covers them:

```
receipt = { jobId, farmerId, tier, trustScore, creditRiskScore,
            recommendedLimitKes, confidence, issuedAt }
```

`agent_jobs.result` already stores the full profile JSON, so the audit trail (lender, farmer, tier, both Cardano txs, explorer URLs, AI reasoning) is preserved automatically.

### 3. Loan review UI — `src/routes/lender.applications.$id.tsx`
Replace the current "Masumi paywall reveal" card body with a richer post-payment view:

- **Headline strip**: Trust Score · Credit Risk · Recommended Limit (KES) · Confidence (% bar)
- **AI Risk Explanation** panel (gemini-generated narrative)
- **Positive / Negative factors** as two-column chip lists
- **Signed assessment footer**: Masumi job id, both Cardano explorer links (lender→AgriTrust, AgriTrust→Climate agent), HMAC signature (truncated, copy button), "Reviewed by loan officer" badge that flips once the officer clicks any decision button
- Decision buttons (Approve / Approve w/ conditions / Request info / Reject) stay where they are and now record the agent job id alongside the decision note.

### 4. Acceptance

- Clicking "Invoke AgriTrust Agent · Premium" on a pending application shows the five required fields plus AI explanation within ~5s.
- The HMAC signature verifies against the displayed receipt JSON.
- Refreshing the page rehydrates the same assessment from `agent_jobs` (no double-charge).
- Featherless gateway 429/402 falls back gracefully with a visible "fallback reasoning" tag.
- Build and typecheck pass; no new env vars.

## Out of scope

- Farmer profile screen, dedicated audit-trail page, swapping Lovable AI for Featherless (can be added later by changing one model id).
